# Project Management Rules and Workflow

## Project Manager (PM) Responsibilities:

- Review documents in the scope and ensure new tasks are created according to the existing scope.
- Create new tasks .md file only in the To-Do list Folder.
- Review existing source code and estimate changes required.
- Provide instructions on where changes are required and their impact.

## Developer Responsibilities:

- Pick tasks .md from the To-Do folder.
- Move tasks .md to In Progress folder and start working on them.
- Move tasks .md to Completed folder when finished.
- Follow instructions and write code that doesn't break the DRY principle.
- Reuse existing code whenever possible.
- Ensure code is maintainable and can be worked on by multiple developers.
- Run lint checks on files worked on and fix errors.

## Workflow:

1. PM creates new tasks in the To-Do folder based on the scope.
2. Developer picks tasks from the To-Do folder and moves them to In Progress folder.
3. Developer works on the task and moves it to Completed folder when finished.
4. PM reviews the completed task and provides feedback.
